# KUMASHA 短時間ARG 運用メモ

シーシャ屋「KUMASHA」で実施する、入口に一番近い席を起点にした短時間ARGです。  
来店客は掠れたMIXカードと男性客の手書きメモを読み、カード裏面のXアカウントへ移動します。Xの最新3投稿からフレーバーを復元し、プロフィールURLの合言葉入力ページで正解すると、元スタッフ「あかり」のエピローグが開きます。

## サイト構成

- `/`：あかりの個人日記風の合言葉入力ページ
- `/print/`：印刷用MIXカード、カード裏面、男性客の手書きメモ

## 起動手順

```bash
npm install
npm run dev
```

開発サーバー起動後、表示されたURLを同じ端末のブラウザで確認してください。印刷物は `/print/` を開いて、ブラウザの印刷機能または画面内の印刷ボタンから出力します。

## スマートフォン公開手順

このサイトはバックエンドなしの静的サイトです。スマートフォンでいつでも開けるようにするには、`dist` フォルダの中身を静的ホスティングへアップロードしてください。

```bash
npm run build
```

おすすめの無料公開方法：

1. Netlify Drop を開く。
2. `npm run build` で生成された `dist` フォルダをアップロードする。
3. 発行されたURLをXプロフィールURLに設定する。
4. 必要ならNetlify側でURL名を変更する。

GitHub Pages、Cloudflare Pages、Vercelなどでも同じように公開できます。ビルド出力は相対パスにしているため、サブディレクトリ配信でも壊れにくい構成です。

### GitHub Pagesで公開する場合

このリポジトリにはGitHub Pages用のActions設定を入れています。

1. GitHubで新しいリポジトリを作る。
2. このプロジェクト一式を `main` ブランチにpushする。
3. GitHubのリポジトリ画面で `Settings` → `Pages` を開く。
4. `Source` を `GitHub Actions` にする。
5. `Actions` タブで `Deploy Pages` が成功したら、表示されたURLをXプロフィールURLに設定する。

## 物理小道具の配置

1. `/print/` から「MIXカード表面」「MIXカード裏面」「男性客の手書きメモ」を印刷する。
2. MIXカード表面と裏面をカードとして使えるように切り出す。
3. 入口に一番近い席に、掠れたMIXカードと男性客の手書きメモを置く。
4. XプロフィールURLに、公開した合言葉入力ページURLを設定する。

## Xアカウント情報

実際のXアカウントは店舗側で作成してください。IDは仮で `@akari_aftershift` とします。取得できない場合は変更して構いませんが、その場合はMIXカード裏面のIDも同じものに変更してください。

表示名：

```text
あかり
```

プロフィール：

```text
前に水タバコ屋で働いてました。
味のメモと日記。
DMあまり見ません。
```

プロフィールURL：

```text
このサイトの合言葉入力ページURLを設定する。
```

最新3投稿は以下にしてください。この3投稿の文章中に、メニューに載っているフレーバー名が一つずつ隠れています。

```text
コンビニのでも十分うれしい。

```

```text
駅前でロイヤルティーって書いてある缶を見つけた。
```
